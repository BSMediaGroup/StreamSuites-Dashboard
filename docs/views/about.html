About StreamSuites

Current Phase

StreamSuites is a modular automation runtime for multi-platform livestream operations. It favors deterministic, schema-driven automation over platform-native bot UIs and keeps orchestration centralized in the runtime so connectors for Rumble, Twitch, YouTube, and Discord stay coordinated without per-platform divergence.

The dashboard you are viewing is deliberately static and read-only. It hydrates from exported JSON bundles, mirrors runtime contracts, and never issues live commands or API calls. Runtime state remains authoritative; the dashboard is a separate artifact for visibility and configuration drafting only.

Early-stage / alpha: contracts and exports are still being aligned, the dashboard stays documentation-grade, and runtime drops remain the source of truth for any visible data.

Architecture & Separation of Concerns

  • StreamSuites Unified Runtime: Executes automation, enforces limits and quotas, connects to platforms, and remains the sole source of truth for chat triggers, jobs, and live state.
  • StreamSuites Dashboard (this repo): Static HTML/CSS/JS that runs in-browser, stores configuration locally, and visualizes runtime-exported state. No backend calls or live control paths are present.
  • Platform Clients: Per-platform adapters (Discord, Twitch, YouTube, Rumble, Twitter) live alongside the runtime and consume exported configuration. The dashboard reflects their expected contracts only.

Configuration, enforcement, observability, and execution are intentionally decoupled to preserve safety, auditability, and deterministic behavior.

Milestone: Rumble SSE Chat Ingestion

  • Rumble chat now uses the canonical SSE endpoint: https://web7.rumble.com/chat/api/chat/{CHAT_ID}/stream.
  • SSE stream emits init and messages events for reliable, scalable ingestion.
  • Runtime export alignment is in progress; dashboard hydration will follow once JSON snapshots are published.
  • Dashboard language stays neutral to polling vs SSE and renders runtime-exported state only.

No deprecated Rumble endpoints are assumed. The dashboard will remain read-only and rely on runtime-produced data drops.

  • Import and export trigger definitions

Jobs

Placeholder & Scaffolded Modules

  • Clips, Polls, Rate Limits, Settings, Chat Replay
  • Platform views for Rumble, YouTube, Twitch, Twitter, Discord (static visibility only)

Current Limitations & Deferred Features

  • No live platform connectivity from the dashboard; runtime exports remain authoritative.
  • No authentication, permissions, or role gating in this phase.
  • Chat Replay, Rate Limits, Clips, and Polls are UI placeholders pending runtime support.
  • Platform pages are read-only scaffolds; they do not initiate API calls or bot actions.
  • No backend APIs or live data bindings are assumed or invoked.

Dashboard Modes (Creator vs Admin)

The layout reserves space for future mode differentiation without enforcing it today:

  • Creator Mode: personal automation setup and visibility.
  • Admin Mode: cross-creator observability and system controls.

Mode-aware behavior is deferred until authentication and authorization are introduced.

Planned Modules

Levelling System

  • Tracks live chat participation per user
  • Each message contributes configurable score values
  • Scores accrue per registered creator
  • Long-term goal: unify viewer identities across Rumble, Twitch, and YouTube using regex/exact name matching
  • Where successful, a user’s scoreboard reflects their true global score for that creator

Status: Planned / Experimental — Implementation: Runtime-owned (placeholder only)

Fake Gambling / Game Mechanics

  • Purely fictional, no real money
  • Text-triggered chat games (e.g. !steal, !loot @user)
  • Randomized success/failure outcomes
  • Failed attempts may incur in-game penalties
  • Randomness may be true RNG or pattern-based as long as it convincingly appears random
  • All balances and tallies are in-game only

Status: Planned — Implementation: Runtime-owned (placeholder only)

  • Detection may be regex-based or platform-native
  • Monthly gifter scoreboard visualized as a pie chart

  • Scoreboard resets each calendar month
  • Monthly winner is publicly visible
  • Registered creators can view and award winners

StreamSuites Unified Runtime
68% · Contracts Locked + Quota Enforcement

The unified runtime is the automation engine that enforces quotas, schedules jobs, and coordinates platform adapters. Its goal is deterministic execution of the full StreamSuites feature set. Contracts and enforcement are already locked in the runtime; remaining effort is incremental tuning rather than structural rewrites.

Administrator Dashboard UI
63% · ConfigState + Telemetry Snapshots

Admin panels present read-only snapshots for jobs, triggers, quotas, and telemetry in the same shapes the runtime exports. The intent is a reliable visibility layer that stays aligned with ConfigState and telemetry snapshots. It is integrated with the runtime contracts but remains static and non-interactive by design.

Creator Dashboard UI
45% · Runtime Contracts Available

Creator-facing flows surface profiles, platform toggles, and trigger scaffolding in a schema-aligned layout. The vision is to let creators assemble export-ready bundles without touching the runtime directly. The current UI is functional scaffolding tied to the schemas; it renders cleanly but still waits on routine runtime exports for hydration.

Discord Client
45% · Foundation + Heartbeat Plan

The Discord adapter targets bot presence, notifications, and command parity with other platforms. It is meant to expose heartbeat and connection signals while staying in lockstep with runtime quotas. Foundations and heartbeat planning exist, but export alignment is still underway and carefully staged.

Rumble Chat SSE Pipeline
50% · Architecture Solved (SSE) · Export alignment pending

Rumble chat now ingests via the canonical SSE stream with a defined message model. The target is reliable livestream chat handling that exports stable payloads for the dashboard. Runtime ingestion is implemented with the solved architecture, but dashboard hydration still depends on published exports.

Twitch Chat Runtime
30% · Foundation + Planned runtime hydration

The Twitch runtime path is designed around IRC ingestion, trigger matching, and export-ready normalization. It should surface chat state and automation outcomes once finalized. Today it carries foundational schemas and message mapping plans, pending stable exports to drive the dashboard.

YouTube Chat Runtime
30% · Scaffold; runtime exports planned

YouTube support is scoped to ingest live chat, align payloads with runtime schemas, and export snapshots the dashboard can render. The end-state is parity with other platforms for triggers and moderation surfaces. Current progress is scaffold-only with defined JSON shapes and UI shells awaiting runtime drops.

Twitter Client / Integration
5% · Concept / Deferred

Twitter integration is purely conceptual, aimed at future ingestion of mentions, DMs, or chat-like events. The intent is a moderation and announcement surface once platform rules allow it. No scaffolding or exports exist yet; the dashboard only reserves a placeholder.

Cross-Platform Chat Abstraction
25% · Planned normalization layer

A cross-platform layer will normalize chat events across Rumble, Twitch, and YouTube so triggers, scoreboards, and moderation tools work uniformly. The desired outcome is consistent payloads regardless of source. Draft normalization rules exist, but runtime enforcement and exports are still early.

Telemetry Export Surface
20% · Planned runtime health export

Telemetry exports are intended to expose health, quota, and heartbeat signals the dashboard can graph without live calls. The goal is passive observability sourced from runtime drops. Scaffolding exists for the feeds, but no active exports are wired in yet.

Dashboard ↔ Runtime Schema Contract
25% · Planned contract lock once exports publish

The schema contract defines exactly how the dashboard consumes exported JSON for creators, platforms, and telemetry. It should guarantee deterministic imports/exports once finalized. Current work aligns ConfigState with runtime expectations, with final locking pending published export shapes.

Unified Live Chat Replay Module
30% · Runtime signals available

Chat replay is meant to store and rehydrate message archives for post-stream review across platforms. The UI is staged to visualize those transcripts alongside metadata. Runtime signals exist, but stabilized export streams are still needed before artifacts can render.

WinForms GUI (Admin & System Controls)
20% · Planning

A WinForms control surface is planned for operators who prefer a desktop UI. It is intended to mirror admin views and telemetry without bypassing runtime safety. Planning and layout considerations are in place, but no runtime coupling or exports are implemented yet.

Clips Module
48% · Runtime clipping (alpha)

The clips module automates livestream clipping, encodes segments with FFmpeg inside the runtime, and marches each clip through deterministic lifecycle states. Finished clips upload to the default StreamSuites Rumble channel today; future paid tiers will let operators set custom destinations. The dashboard surfaces every clip, including queued, pending, and failed jobs, reflecting runtime truth. Current state: implemented in the runtime, visible in the dashboard, early-stage alpha quality, and expected to iterate.

Levelling System
10% · Planned / Experimental

Levelling is envisioned as a fictional progression track where chat participation earns experience toward ranks or perks. It should eventually respect cross-platform identity mapping once available. This remains early ideation with no runtime scaffolding beyond concept notes.

Fake Gambling / Game Mechanics
10% · Planned

The game mechanics are lightweight chat commands (e.g., !steal, !loot @user) that award or deduct fictional balances. The intent is playful, zero-stakes engagement surfaced in dashboards only. It stays in the planned phase with placeholder UI and no runtime enforcement.

Paid Interaction Detection
15% · Planned / Design In Progress

Paid interaction detection will surface gifted subscriptions and paid shout-outs (Rumble Rants, Super Chats, etc.) with monthly rollups. The dashboard aims to visualize those exports alongside leaderboards. Design work is underway, but runtime feeds are not yet finalized.

Mechanics are not finalized; implementation details remain pending and will follow runtime design updates.
