About StreamSuites

What is StreamSuites?

StreamSuites is a modular, multi-platform livestream automation system designed to give creators structured, deterministic control over how their live streams interact with chat, generate clips, run polls, and publish content across platforms.

The system is intentionally split into two major components:

  • StreamSuites Runtime — the execution engine that connects to livestream platforms, listens to chat, evaluates rules, and performs actions.
  • StreamSuites Dashboard — a static, browser-based configuration and inspection surface that produces runtime-ready configuration.

This separation is deliberate. Configuration and execution are decoupled to ensure safety, clarity, portability, and auditability.

Design Philosophy

  • No magic: all behavior is defined by explicit configuration.
  • No frameworks: minimal dependencies, predictable behavior.
  • Deterministic execution: identical input produces identical output.
  • Read-only safety: dashboards never directly control live bots.
  • Platform-agnostic core: platform logic lives at the edges.

StreamSuites is intentionally designed to be boring in the best sense: inspectable, debuggable, and resistant to platform churn.

System & Repository Topology

  • StreamSuites (Runtime repository): Python-based execution environment that consumes exported configuration and performs live automation.
  • StreamSuites Dashboard (this repository): Static HTML, CSS, and vanilla JavaScript served via GitHub Pages or embedded as an iframe (e.g. Wix Studio).

The dashboard stores configuration locally (via localStorage) and exports JSON that is explicitly imported into the runtime.

Dashboard → JSON Export → Runtime Import → Live Execution

Current Dashboard Features

Overview

  • High-level system and storage status indicators
  • Configuration summary counts
  • Read-only visibility into stored configuration

Creators

  • Create and edit creator profiles
  • Platform enablement (e.g. Rumble)
  • Schema-complete placeholders required by the runtime
  • Import and export creators as JSON

Chat Triggers

  • Define trigger phrases and match modes
  • Configure responses and cooldowns
  • Import and export trigger definitions

Jobs

  • Read-only visibility into job statistics
  • Breakdowns by status, type, and creator

Placeholder & Scaffolded Modules

  • Clips
  • Polls
  • Rate Limits
  • Settings
  • Chat Replay
  • Platform-specific views (Rumble, YouTube, Twitch, Twitter, Discord)

Planned & Future Features

  • Chat-triggered clip generation with timing rules
  • Live polls driven by chat voting
  • Per-creator and per-platform rate limiting
  • Historical chat replay and visualization
  • Multi-platform posting and automation rules
  • Expanded job diagnostics and observability
  • Role-aware dashboard presentation (Creator vs Admin)

Features are staged intentionally and may evolve as platforms and constraints change.

Creator vs Admin Dashboard Modes

StreamSuites is being designed with a future dual-mode dashboard in mind:

  • Creator Mode: Focused on configuring personal automation behavior.
  • Admin Mode: Expanded visibility and control across multiple creators and systems.

At present, all features are visible and no authentication or permission system is enforced. The UI is structured to support future role indication without introducing premature complexity.

Explicit Non-Goals

  • No live bot control from the dashboard
  • No authentication or account system at this stage
  • No opaque automation or hidden behavior
  • No reliance on third-party frontend frameworks

Project Status

StreamSuites is under active development. This dashboard represents the canonical configuration surface for the system and is expected to evolve alongside the runtime as new capabilities are added.

This page exists to provide continuity, orientation, and shared understanding as the project grows.
