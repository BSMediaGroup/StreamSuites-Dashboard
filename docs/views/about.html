About StreamSuites

What is StreamSuites?

StreamSuites is a modular, multi-platform livestream automation system designed to give creators structured, deterministic control over how their live streams interact with chat, generate clips, run polls, and publish content across platforms.

The project is intentionally split into two major components:

  • StreamSuites Runtime — the backend execution engine that connects to livestream platforms, listens to chat, and performs actions.
  • StreamSuites Dashboard — a static, browser-based control surface used to configure, inspect, and export runtime-ready configuration.

This separation ensures clarity, safety, and portability: configuration and execution are deliberately decoupled.

Design Philosophy

  • No magic: all configuration is explicit, inspectable, and exportable.
  • No frameworks: minimal dependencies, predictable behavior.
  • Deterministic behavior: the same input always yields the same output.
  • Read-only safety: the dashboard never controls live bots directly.
  • Platform-agnostic core: platform specifics live at the edges.

StreamSuites is designed to be boring in the best possible way: stable, inspectable, and resilient to platform changes.

High-Level Architecture

  • Dashboard (this app): Static HTML, CSS, and vanilla JavaScript. Runs on GitHub Pages or inside an iframe (e.g. Wix Studio). Stores configuration in localStorage.
  • Runtime: A separate execution environment (Python-based) that consumes exported JSON and performs live automation.
  • Configuration Flow: Dashboard → JSON Export → Runtime Import → Live Execution

The dashboard is intentionally incapable of directly controlling live streams. This is a safety and stability guarantee.

Current Dashboard Features

Overview

  • High-level system status indicators
  • Configuration summary counts
  • Read-only visibility into stored configuration

Creators

  • Create and edit creator profiles
  • Platform enablement (e.g. Rumble)
  • Schema-complete placeholders for runtime requirements
  • Import/export creators as JSON

Chat Triggers

  • Define trigger phrases and match modes
  • Configure responses and cooldowns
  • Import/export trigger definitions

Jobs

  • Read-only visibility into job statistics
  • Breakdowns by status, type, and creator

Placeholder Modules

  • Clips
  • Polls
  • Rate Limits
  • Settings
  • Chat Replay
  • Platform-specific views (Rumble, YouTube, Twitch, Twitter, Discord)

Planned & Future Features

  • Chat-triggered clip generation with timing rules
  • Live polls driven by chat voting
  • Per-creator and per-platform rate limiting
  • Historical chat replay and visualization
  • Multi-platform posting rules
  • Role-aware dashboard presentation (Creator vs Admin)
  • Expanded job inspection and diagnostics

Planned features are intentionally staged and may evolve as platforms and requirements change.

Creator vs Admin Dashboard Modes

StreamSuites is being designed with a future dual-mode dashboard in mind:

  • Creator Mode: Focused on configuring personal automation behavior.
  • Admin Mode: Expanded visibility and control across multiple creators and systems.

At present, all features are visible and no authentication or permission system is enforced. This dashboard is UI-ready for future role indication without introducing premature complexity.

Explicit Non-Goals

  • No live bot control from the dashboard
  • No authentication or account system at this stage
  • No opaque automation or hidden behavior
  • No reliance on third-party frontend frameworks

Project Status

StreamSuites is under active development. This dashboard represents the canonical configuration surface for the system and is expected to evolve alongside the runtime as new capabilities are added.

This page exists to provide continuity and orientation as the project grows.
