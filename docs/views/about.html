About StreamSuites

Version: StreamSuites™ v0.2.0-alpha • Owner: Daniel Clancy • Copyright: © 2025 Brainstream Media Group

Current Phase

StreamSuites is a modular livestream automation platform built around an enforcement-first, deterministic runtime. All orchestration, scheduling, quotas, and state transitions are owned by the runtime so behavior remains consistent across Rumble, Twitch, YouTube, and Discord without platform-specific drift.

The dashboard and public-facing pages are intentionally passive surfaces. They render runtime-exported snapshots, allow controlled configuration authoring, and expose read-only public galleries for clips, polls, tallies, and scoreboards without issuing live commands or API calls.

The platform is rolling out module by module. Runtime remains authoritative. The dashboard stays static by design. Public pages reflect shipped exports rather than live data pulls.

Architecture & Separation of Concerns

  • StreamSuites Unified Runtime: Executes automation, enforces quotas, schedules jobs, aggregates signals, and exports deterministic snapshots. It is the sole source of truth.
  • StreamSuites Dashboard (this repo): A static HTML/CSS/JS application that runs fully client-side. It visualizes runtime exports, drafts configuration, and exposes public gallery views without any backend or control-plane authority.
  • Public Pages: Standalone gallery and detail pages for clips, polls, tallies, and scoreboards. These pages are read-only and hydrate exclusively from runtime-produced JSON.

Configuration, enforcement, observability, and execution are deliberately decoupled to preserve safety, auditability, and deterministic behavior.

Milestone: Rumble SSE Chat Ingestion

  • Rumble chat now ingests via the canonical SSE endpoint: https://web7.rumble.com/chat/api/chat/{CHAT_ID}/stream
  • The stream emits init and messages events for reliable, scalable ingestion.
  • Runtime-normalized exports now feed dashboards, public galleries, and downstream aggregation modules.
  • The dashboard remains transport-agnostic and renders exported state only.

Deprecated polling-based endpoints are not assumed. All ingestion details live exclusively in the runtime.

Dashboard Capabilities (Static)

Overview

  • High-level system, storage, and export visibility
  • Snapshot-based telemetry when runtime drops are present
  • Public galleries for clips, polls, tallies, and scoreboards

Creators

  • Creator profile authoring aligned to runtime schemas
  • Platform enablement and notes fields
  • Deterministic import/export as JSON

Chat Triggers

  • Trigger phrase definition, match modes, and cooldowns
  • Exported trigger catalogs rendered read-only

Data & Signals

  • Read-only visibility into runtime databases and signal feeds
  • Paginated, searchable tables for clips, polls, tallies, scoreboards, triggers, and creators
  • Designed as an observability and debugging surface

Current Limitations

  • No live runtime control from the dashboard
  • No authentication or role enforcement yet
  • Public pages are strictly read-only
  • No backend APIs are assumed or invoked

StreamSuites Unified Runtime
75% · Contracts Locked + Exports + Aggregation

The unified runtime is the authoritative automation engine. It enforces quotas, schedules jobs, ingests platform chat, aggregates signals, and emits deterministic exports that drive all dashboard views and public galleries. Core contracts are locked; remaining work focuses on deeper orchestration, additional platform parity, and expanded module mechanics.

Administrator Dashboard UI
70% · ConfigState + Data & Signals + Public Pages

The administrator dashboard provides a static, read-only observability surface for runtime exports, configuration state, and public gallery parity. It is intentionally non-interactive and designed for debugging, inspection, and system-wide visibility without issuing live commands.

Creator Dashboard UI
56% · Schema-Aligned Authoring + Export Drafting

Creator-facing flows expose profiles, platform toggles, triggers, and module scaffolding in a schema-aligned layout. The dashboard allows creators to assemble deterministic, export-ready configurations without interacting with the runtime directly.

Discord Client
45% · Foundation + Heartbeat Plan

The Discord adapter targets bot presence, notifications, and command parity with other platforms. It is meant to expose heartbeat and connection signals while staying in lockstep with runtime quotas. Foundations and heartbeat planning exist, but export alignment is still underway and carefully staged.

Rumble Chat SSE Pipeline
60% · SSE Architecture + Exports Published

Rumble chat now ingests via the canonical SSE stream with a defined message model. The runtime exports stable payloads for dashboards and public pages, and the pipeline is wired into quotas and enforcement. Remaining work focuses on deeper moderation features and extended telemetry.

Twitch Chat Runtime
40% · IRC Foundation + Export Mapping

The Twitch runtime path is designed around IRC ingestion, trigger matching, and export-ready normalization. It now carries executable ingestion and normalization plans, with exports being shaped to match dashboard and public page expectations. Next steps are quota enforcement parity and richer telemetry.

YouTube Chat Runtime
35% · Scaffold + Message Mapping Drafts

YouTube support is scoped to ingest live chat, align payloads with runtime schemas, and export snapshots the dashboard can render. Message mapping drafts are defined for parity with other platforms. Current progress is scaffold plus partial ingestion paths, awaiting stabilized exports for dashboards.

Twitter Client / Integration
5% · Concept / Deferred

Twitter integration is purely conceptual, aimed at future ingestion of mentions, DMs, or chat-like events. The intent is a moderation and announcement surface once platform rules allow it. No scaffolding or exports exist yet; the dashboard only reserves a placeholder.

Chat Triggers
60% · Enforcement + Exported Responses

Trigger definitions run inside the runtime with deterministic matching, quotas, and cooldown enforcement. Exports now include trigger catalogs and response shapes that the dashboard and public pages render read-only. Next steps focus on broader platform parity and expanded moderation actions.

Cross-Platform Chat Abstraction
30% · Normalization Drafts + Export Targets

A cross-platform layer will normalize chat events across Rumble, Twitch, and YouTube so triggers, scoreboards, and moderation tools work uniformly. The desired outcome is consistent payloads regardless of source. Draft normalization rules exist, but runtime enforcement and exports are still early.

Telemetry Export Surface
30% · Draft Health + Quota Exports

Telemetry exports are intended to expose health, quota, and heartbeat signals the dashboard can graph without live calls. The goal is passive observability sourced from runtime drops. Scaffolding exists for the feeds, but no active exports are wired in yet.

Dashboard ↔ Runtime Schema Contract
55% · Deterministic Exports + Public Gallery Alignment

The schema contract defines exactly how the dashboard consumes exported JSON for creators, platforms, public galleries, and telemetry. It now guarantees deterministic imports/exports for current modules and keeps expanding alongside new runtime drops.

Unified Live Chat Replay Module
35% · Runtime Signals + Export Shapes

Chat replay is meant to store and rehydrate message archives for post-stream review across platforms. The UI is staged to visualize those transcripts alongside metadata. Runtime signals exist, but stabilized export streams are still needed before artifacts can render.

WinForms GUI (Admin & System Controls)
20% · Planning

A WinForms control surface is planned for operators who prefer a desktop UI. It is intended to mirror admin views and telemetry without bypassing runtime safety. Planning and layout considerations are in place, but no runtime coupling or exports are implemented yet.

Scoreboards
42% · Aggregation Shapes + Public Gallery

Scoreboards aggregate trigger outputs and interaction tallies into public-facing leaderboards. Export shapes and public gallery shells exist, with runtime-side aggregation cadence and reset logic still being finalized.

Clips Module
62% · Runtime Clipping + Public Gallery

The clips module automates livestream clipping, encodes segments with FFmpeg inside the runtime, and marches each clip through deterministic lifecycle states. Finished clips upload to the default StreamSuites Rumble channel today with exports that power dashboard and public gallery/detail pages. Current state: implemented in the runtime, visible across dashboard and public surfaces, with destination flexibility and moderation tooling still expanding.

Polls Module
58% · Runtime Polling + Public Gallery

Polls execute in the runtime with strict quotas and deterministic close conditions. Dashboard and public pages show read-only ballots, results, and detail pages sourced from runtime exports. Remaining work focuses on broader platform coverage, richer scheduling, and export cadence tuning.

Tallies Module
48% · Aggregation + Public Gallery + Detail Pages

Tallies are programmatic, chat-driven aggregations that count events over weekly, monthly, or custom windows. Public gallery and detail pages are live and read-only, driven by runtime exports. Runtime-side scheduling, trigger coupling, and enforcement logic continue to mature.

Levelling System
10% · Planned / Experimental

Levelling is envisioned as a fictional progression track where chat participation earns experience toward ranks or perks. It should eventually respect cross-platform identity mapping once available. This remains early ideation with no runtime scaffolding beyond concept notes.

Fake Gambling / Game Mechanics
10% · Planned

The game mechanics are lightweight chat commands (e.g., !steal, !loot @user) that award or deduct fictional balances. The intent is playful, zero-stakes engagement surfaced in dashboards only. It stays in the planned phase with placeholder UI and no runtime enforcement.

Paid Interaction Detection
15% · Planned / Design In Progress

Paid interaction detection will surface gifted subscriptions and paid shout-outs (Rumble Rants, Super Chats, etc.) with monthly rollups. The dashboard aims to visualize those exports alongside leaderboards. Design work is underway, but runtime feeds are not yet finalized.

Mechanics are not finalized; implementation details remain pending and will follow runtime design updates.
