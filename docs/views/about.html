About StreamSuites

Current Phase

StreamSuites is in a stabilization phase focused on deterministic configuration, static visibility, and platform-aligned scaffolds. The dashboard remains a static, read-only surface that mirrors the runtime contracts without driving live automation.

Scope for this repo: documentation-grade UI, local storage inspection, and JSON exports only.

Architecture & Separation of Concerns

  • StreamSuites Unified Runtime: Executes automation, connects to platforms, and remains the source of truth for chat triggers, jobs, and live state.
  • StreamSuites Dashboard (this repo): Static HTML/CSS/JS that runs in-browser, stores configuration locally, and exports JSON for the runtime. No backend calls or live control paths are present.
  • Platform Clients: Per-platform adapters (Discord, Twitch, YouTube, Rumble, Twitter) live alongside the runtime and consume exported configuration. The dashboard only reflects their expected contracts.

Configuration, observability, and execution are intentionally decoupled to preserve safety and auditability.

Dashboard Capabilities (Static)

Overview

  • High-level storage status and configuration counts
  • Snapshot visibility for creators, triggers, and jobs

Creators

  • Create and edit creator profiles
  • Platform enablement fields aligned with runtime schemas
  • Import and export creators as JSON

Chat Triggers

  • Define trigger phrases, match modes, responses, and cooldowns
  • Import and export trigger definitions

Jobs

  • Read-only statistics and breakdowns

Placeholder & Scaffolded Modules

  • Clips, Polls, Rate Limits, Settings, Chat Replay
  • Platform views for Rumble, YouTube, Twitch, Twitter, Discord (static visibility only)

Current Limitations & Deferred Features

  • No live platform connectivity from the dashboard; runtime exports remain authoritative.
  • No authentication, permissions, or role gating in this phase.
  • Chat Replay, Rate Limits, Clips, and Polls are UI placeholders pending runtime support.
  • Platform pages are read-only scaffolds; they do not initiate API calls or bot actions.
  • No backend APIs or live data bindings are assumed or invoked.

Dashboard Modes (Creator vs Admin)

The layout reserves space for future mode differentiation without enforcing it today:

  • Creator Mode: personal automation setup and visibility.
  • Admin Mode: cross-creator observability and system controls.

Mode-aware behavior is deferred until authentication and authorization are introduced.

Project Status

The dashboard tracks the StreamSuites runtime but does not replace it. Each view is intentionally static, mapping to stable DOM IDs and schemas so runtime exports can hydrate them later without restructuring the UI.
