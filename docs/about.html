About StreamSuites™

Creator-first

StreamSuites pairs a GitHub Pages-friendly dashboard with public runtimes so creators can manage jobs, clips, and monetization while viewers see only the polished, static surfaces. We keep routing simple, pages lightweight, and updates predictable.

This public area mirrors the dashboard tone but never reuses dashboard routing or logic. Everything here is static, cache-safe, and ready for embeds.

Runtime vs. Dashboard

The runtime powers automations, platform hooks, and delivery. The dashboard remains a static admin shell that controls those runtimes without exposing secrets. Public pages sit alongside, offering safe previews and documentation.

  • Dashboard stays noindex/nohistory and ships static assets only.
  • Runtime endpoints are isolated, hardened, and versioned.
  • Public pages reuse the theme without importing dashboard routers.

Platform Coverage

StreamSuites tracks the surfaces creators care about most.

  • Streaming: Twitch, YouTube, Kick (monitoring + scheduling)
  • Video: Rumble, YouTube Live archives, VOD safelists
  • Community: Discord delivery, chat triggers, loyalty tallies

Reliability

We favor static fallbacks, explicit routes, and small payloads. Version badges on every public page mirror docs/version.json so visitors know what build they are seeing.

  • Manual changelog plus visual roadmap
  • Static navigation baseline across pages
  • Dark glass containers that keep readability intact
