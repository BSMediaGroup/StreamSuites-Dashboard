# StreamSuites-Dashboard v0.4.2-alpha

`StreamSuites-Dashboard v0.4.2-alpha` separates Alerts from Analytics as distinct admin workspaces, hardens dashboard hydration after the geo rollout, and brings alert authoring closer to the runtime-owned contract without moving backend authority into the frontend.

## 🚨 Alerts and Analytics Separation

- Split Alerts into its own dedicated admin route and sidebar destination instead of treating it as a narrow Analytics subpanel.
- Restored Analytics as its own reporting page with the geo map and related reporting surfaces preserved.
- Continued shell and sidebar cleanup so the current route structure reads cleanly across overview, analytics, alerts, and the other admin views.

## 🧠 Alert Authoring and Rule Editing

- Expanded alert authoring with tokenized placeholders, richer geo and request variables, `user_code`, visitor IP support, and page-path deep links where a canonical URL exists.
- Replaced the old single-surface selector with backward-compatible multi-surface filtering that better matches the backend alert-rule model.
- Added a live multi-surface preview matrix so Desktop, Browser, and Plain alert renders can be compared without changing stored template syntax.

## 🌍 Hydration Hardening and Reporting Polish

- Hardened admin hydration and state loading after the geo analytics regression so one failing section is less likely to blank the whole dashboard.
- Improved location formatting across analytics and alerts so city, region, and country fallback labels render more cleanly.
- Added the FindMeHere Directory surface to analytics and alerting views so directory traffic is no longer merged into generic public activity.
